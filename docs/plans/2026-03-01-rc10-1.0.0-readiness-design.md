# rc10 / 1.0.0 Readiness Design

## Goal

Comprehensive white-glove audit of Nexus from rc9 → rc10, with rc10 as the validation
release before 1.0.0 final.

## Context

- Current version: `1.0.0rc9`
- CHANGELOG: entries through rc6 only — rc7/rc8/rc9 missing
- 12 commits since rc7 tag unaccounted for in CHANGELOG
- Docs likely stale: cli-reference, storage-tiers, getting-started, contributing
- Plugin docs (nx/README.md) may not reflect new skills/hooks added in rc7–rc9
- Release path: audit + fix → rc10 → validate → 1.0.0 (next session)

## Audit Domains

| Domain | Artifact | Known Gap |
|--------|----------|-----------|
| CHANGELOG | `CHANGELOG.md` | rc7/rc8/rc9 entries missing |
| CLI reference | `docs/cli-reference.md` | scratch, workflow, T1, flag removals post-rc6 |
| Storage tiers | `docs/storage-tiers.md` | T1 HTTP server not documented |
| Getting started | `docs/getting-started.md` | May be stale |
| Contributing | `docs/contributing.md` | Release process current? |
| Plugin README | `nx/README.md` | New skills/hooks/agents added in rc7–rc9 |
| Plugin registry | `nx/registry.yaml` | Agent descriptions current? |
| Source code | `src/nexus/` | TODOs, FIXMEs, error message quality |
| Packaging | `pyproject.toml` | Metadata correct for 1.0.0 milestone? |
| CI/CD | `.github/workflows/release.yml` | CHANGELOG extraction matches actual format? |

## Process

1. **Substantive critic pass** — single agent, comprehensive across all domains above
2. **Triage** — review findings together, approve which to fix for rc10
3. **Fix** — fix approved findings in priority order, one commit per domain
4. **rc10 release** — bump version, write rc10 CHANGELOG entry, tag, push, PR, merge

## Success Criteria

- [ ] CHANGELOG has accurate entries for rc7, rc8, rc9, rc10
- [ ] `docs/cli-reference.md` matches actual `nx --help` tree
- [ ] `docs/storage-tiers.md` documents T1 HTTP server architecture
- [ ] `nx/README.md` reflects all current skills, hooks, agents
- [ ] No TODO/FIXME left in production code paths
- [ ] All first-run error messages are actionable
- [ ] `pyproject.toml` metadata correct and complete
- [ ] rc10 installable via `pip install conexus==1.0.0rc10`
- [ ] CI green on rc10 tag
