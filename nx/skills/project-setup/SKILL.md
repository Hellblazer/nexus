# Project Setup Skill

Delegates to the **project-management-setup** agent (haiku). See [registry.yaml](../../registry.yaml).

## When This Skill Activates

- When starting multi-week projects (>3 weeks)
- When systematic tracking and resumability is required
- When project context tracking is needed
- When knowledge integration infrastructure is required
- When user says "set up project management"

## Agent Invocation

Use the Task tool to invoke **project-management-setup**:

```markdown
## Relay: project-management-setup

**Task**: [what needs to be done]
**Bead**: [ID] or 'none'

### Input Artifacts
- Files: [relevant files]

### Deliverable
PM infrastructure with tracking and context docs

### Quality Criteria
- [ ] nx pm init completed
- [ ] Core T2 documents created
- [ ] Task beads created with dependencies
```

For full relay structure and optional fields, see [RELAY_TEMPLATE.md](../../agents/_shared/RELAY_TEMPLATE.md).

## Infrastructure Created

The agent runs `nx pm init`, which creates four standard T2 entries:

- `METHODOLOGY.md` — Engineering methodology and TDD discipline
- `BLOCKERS.md` — Active blockers, one per line
- `CONTEXT_PROTOCOL.md` — Agent-facing protocol reference
- `phases/phase-1/context.md` — Initial project context, goals, and state

Access via `nx pm status` or `nx memory get --project <name> --title <doc>.md`.

## Success Criteria

- [ ] `nx pm init` completed; `nx pm status` returns project info
- [ ] Core T2 documents created (METHODOLOGY, BLOCKERS, CONTEXT_PROTOCOL, phase-1 context)
- [ ] Epic bead created for project
- [ ] Task beads created with dependencies
- [ ] Ready for strategic-planning to begin

## Agent-Specific PRODUCE

Outputs generated by the project-management-setup agent:

- **nx pm**: Initialized via `nx pm init`; accessible via `nx pm status`
- **T2 memory**: Core PM documents: `METHODOLOGY.md`, `BLOCKERS.md`, `CONTEXT_PROTOCOL.md`, `phases/phase-1/context.md` — created via `nx pm init`
- **T1 scratch**: Interim setup notes via `nx scratch put "Setup note: {observation}" --tags "pm-setup"` (flagged for T2 if worth preserving)

**Session Scratch (T1)**: Agent uses `nx scratch` for ephemeral working notes during the session. Flagged items auto-promote to T2 at session end.
